First attempt at doom-harness. The WASI shim is a
       minimum viable subset; expect bumps (wrong errno on edge cases, framebuffer
       color order, save-game writes that return ENOSYS). Frames are blitted into
       the canvas via env.draw_frame; keys are read from a queue via
       env.get_key; the WAD is served through a fd-backed virtual fs
       (preopen "/").
